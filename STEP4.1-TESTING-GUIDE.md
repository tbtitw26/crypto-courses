## Тест 5: Мультиязычность (2 job'а для EN+AR) (Этап 5)

### Цель
Убедиться, что при выборе двух языков создаются 2 job'а и генерируются 2 PDF.

### Шаги

1. **Сгенерировать Custom Course с двумя языками:**
   - Перейти на `/learn?tab=custom`
   - В форме выбрать оба языка: EN и AR
   - Заполнить остальные поля
   - Нажать "Generate"

2. **Проверить API response:**
   - Открыть DevTools → Network
   - Найти запрос к `/api/custom-course`
   - Проверить response:
     - **Ожидаемый формат:**
       ```json
       {
         "ok": true,
         "jobs": [
           { "jobId": 10, "language": "en" },
           { "jobId": 11, "language": "ar" }
         ],
         "jobId": 10,  // backward-compatible
         "message": "Course generation queued for 2 language(s)..."
       }
       ```
     - `jobs` array содержит 2 элемента ✅
     - Каждый элемент имеет `jobId` и `language` ✅

3. **Проверить базу данных:**
   - Открыть Prisma Studio
   - Найти записи `CustomCourseRequest` с `id = 10` и `id = 11`
   - **Ожидаемый результат:**
     - Запись 10: `language = 'en'` ✅
     - Запись 11: `language = 'ar'` ✅
     - Обе записи имеют `status = 'in_queue'` или `'processing'` ✅

4. **Проверить Inngest Dev Server:**
   - Открыть `http://localhost:8288`
   - Найти 2 runs для `custom_course/requested`:
     - Run 1: `jobId = 10`, `language = 'en'` ✅
     - Run 2: `jobId = 11`, `language = 'ar'` ✅
   - Оба runs должны завершиться успешно ✅

5. **Проверить Supabase Storage:**
   - Открыть Supabase Dashboard → Storage → `course-pdf`
   - Найти 2 PDF файла:
     - `custom/${courseId}-en-10.pdf` ✅
     - `custom/${courseId}-ar-11.pdf` ✅

6. **Проверить frontend:**
   - На странице `/learn?tab=custom` должны отображаться 2 блока статуса:
     - "EN — Status" с прогрессом для jobId=10 ✅
     - "AR — Status" с прогрессом для jobId=11 ✅
   - Каждый блок показывает свой статус и прогресс ✅
   - Когда оба готовы, каждый блок показывает кнопку "Open PDF" ✅

   - Проверить, что создаются 2 job'а
   - Проверить, что генерируются 2 PDF

### Критерии успеха
- ✅ При выборе 2 языков создаются 2 job'а в БД
- ✅ Отправляются 2 Inngest events
- ✅ API возвращает `jobs` array с 2 элементами

## Что делать, если что-то не работает

1. **Pipeline застрял:**
   - Проверить Inngest Dev Server — на каком step остановился
   - Проверить логи Next.js — есть ли ошибки
   - Проверить базу данных — какой статус у job'а

2. **PDF слишком большой:**
   - Проверить, используется ли `COMPACT_CUSTOM_COURSE_JSON_SCHEMA`
   - Проверить, работает ли post-LLM clamp
   - Проверить логи генерации

3. **Не создаются 2 job'а:**
   - Проверить API response — есть ли `jobs` array
   - Проверить базу данных — созданы ли 2 записи
   - Проверить Inngest Dev Server — отправлены ли 2 events

4. **Frontend не показывает несколько блоков:**
   - Проверить DevTools → Console — есть ли ошибки
   - Проверить, что `activeJobs` state содержит несколько job'ов
   - Проверить, что `JobStatusBlock` компонент используется
